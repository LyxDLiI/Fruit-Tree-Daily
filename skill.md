# 果业技术日报 Skill

## 适用场景（触发词）
- 果业日报
- 果业技术日报
- 水果日报
- 发一份日报
- 今日果业行情 / 今日水果价格 / 果业政策更新

当用户要“最新/今天/本周”内容时，必须先读取当前日期，再执行检索。

## 目标
产出一份**当天可执行、信息全面、结构统一**的日报，覆盖：
1. 技术进展（栽培、病虫害、灾害防御、采后）
2. 市场行情（批发价格、指数、产销趋势）
3. 政策动态（国家部委、主产区政策）
4. 季节提醒（按物候期的管理动作）

## 工作流（严格顺序）

### 步骤 0：确定时间上下文
1. 获取当前日期与北京时间（Asia/Shanghai）。
2. 生成三个时间窗：
   - T0：当天
   - T-3：近 3 天
   - T-7：近 7 天（兜底）
3. 报头使用 `YYYY年M月D日`，生成时间使用 `YYYY-MM-DD HH:mm (UTC+8)`。

### 步骤 1：多维检索（至少 6 组）
按下列维度检索；若结果不足，自动扩展同义关键词重试。

1. **技术与种植管理**
   - `果业技术 水果种植 管理 {T0}`
   - `苹果 柑橘 葡萄 香蕉 西瓜 病虫害 防控 {T-3}`
2. **市场价格与指数**
   - `农业农村部 农产品批发价格200指数 水果 {T0}`
   - `全国农产品批发市场 水果价格 监测 {T-3}`
3. **政策与产业动态**
   - `农业农村部 果业 政策 通知 {当月}`
   - `海关总署 水果 进出口 {当月}`
4. **产区与天气灾害联动**
   - `苹果主产区 倒春寒 冻害 预警 {T-3}`
   - `柑橘 葡萄 产区 天气 风险 {T-3}`
5. **科研/推广信息**
   - `果树 研究所 技术要点 {T-7}`
   - `农业技术推广 果树 栽培 培训 {T-7}`
6. **消费与流通（可选增强）**
   - `水果 电商 销售 数据 {T-7}`
   - `水果 冷链 物流 价格 {T-7}`

> 若用户指定品类（如仅苹果），保留全局扫描但在输出中提升该品类优先级。

### 步骤 2：来源优先级与筛选
按以下优先级取材，并为每条信息保留来源与日期：

- P1（最高）：政府与官方机构（农业农村部、国家统计/气象、海关、地方农业农村厅）
- P2：国家级批发市场与行业协会
- P3：高校/科研院所/农技推广站
- P4：主流媒体与行业媒体（仅作补充，需二次交叉验证）

筛选规则：
1. 优先 T0 与 T-3；无结果再用 T-7。
2. 每个板块至少 2 条有效信息。
3. 同一事实多来源重复时合并，保留最权威来源。
4. 无法确认发布时间或来源可信度低的内容剔除。

### 步骤 3：结构化整合
输出前先做“事实卡片”整理：
- 标题（15~28 字）
- 一句话结论（≤50 字）
- 影响对象（品类/区域）
- 建议动作（可执行）
- 来源 + 日期

再按模板生成日报正文。

### 步骤 4：质量检查（发布前）
- [ ] 是否明确写出“今天”对应的绝对日期
- [ ] 是否至少覆盖 4 个板块（技术/市场/政策/季节）
- [ ] 是否至少覆盖 4 个主要水果品类
- [ ] 是否包含可执行建议（如“48小时内完成…/关注…阈值”）
- [ ] 是否标注来源类别与日期
- [ ] 是否在“来源信息”板块直接打印可访问 URL

若任一项不满足，补检索后再输出。

## 生成策略（让内容更“全面且总结到位”）
1. **先横向覆盖，再纵向深入**：先保证四大板块齐全，再补充重点品类深挖。
2. **结论优先**：每条信息先给“结论”，再给“依据与动作”。
3. **数字优先**：有价格、涨跌幅、时间阈值时优先写数字。
4. **区域对比**：至少给出 1 组“主产区 vs 主销区”变化。
5. **风险前置**：天气/病虫害/价格波动风险写在建议前。

## 缺失数据时的降级策略
- 未查到全国统一价格时：使用多个重点市场均价并说明口径。
- 未查到当日政策时：回溯近 7 天并标注“近7日新增”。
- 未查到品类数据时：明确写“暂缺”，给出替代观察指标（如到货量、天气风险）。

## 维护建议
- 每月复盘关键词命中率，替换低命中词。
- 每季度更新“重点品类/产区”名单。
- 若进入关键物候期（花期、膨果期、采收期），将季节提醒提升为日报首段。
